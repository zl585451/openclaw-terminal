# OCT 反馈机器人配置

## 飞书机器人配置

### 1. 创建飞书机器人
1. 打开飞书群
2. 群设置 → 机器人 → 添加机器人
3. 选择"自定义机器人"
4. 复制 Webhook 地址

### 2. 填写配置
```json
{
  "feishu_webhook": "https://open.feishu.cn/open-apis/bot/v2/hook/xxx",
  "github_token": "ghp_xxx",
  "github_repo": "zl585451/openclaw-terminal"
}
```

### 3. 启动机器人
```bash
python scripts/feedback-bot.py
```

### 4. 测试
在群里发送：
```
这个功能好用！
```
或
```
有个 bug，点击按钮没反应
```

机器人会自动创建 GitHub Issue！

## 自动收集的反馈类型

| 类型 | 关键词 | 自动标签 |
|------|--------|---------|
| **Bug 报告** | bug, 问题，错误，报错，崩溃 | `bug` |
| **功能建议** | 建议，希望，想要，功能 | `enhancement` |
| **正面反馈** | 好用，喜欢，不错，赞 | `positive` |
| **负面反馈** | 难用，不好，卡顿，慢 | `negative` |

---

## GitHub Issues 自动创建

**标题格式**: `[情感] 用户反馈：内容摘要`

**内容包含**:
- 反馈原文
- 用户信息（匿名）
- 时间戳
- 情感分析结果
- 自动分类标签

**示例**:
```
[POSITIVE] 用户反馈：这个功能好用！

## 反馈内容
这个功能好用！

## 用户信息
- 用户：张三
- 时间：2026-03-11 20:20
- 情感：positive

## 自动分类
- 类型：用户反馈
- 优先级：待评估
```

## 高级功能（可选）

### 1. 自动回复
```python
# 在 webhook 中添加自动回复
auto_reply = {
    "bug": "收到！我们会尽快修复～",
    "建议": "好建议！已记录～",
    "好评": "谢谢支持！❤️"
}
```

### 2. 反馈统计
每周自动生成反馈报告：
- 收到多少反馈
- 正面/负面比例
- 最常见的问题
- 最受欢迎的功能

### 3. 优先级排序
根据关键词自动标记优先级：
- 🔴 高：崩溃、数据丢失
- 🟡 中：功能问题、卡顿
- 🟢 低：建议、优化
