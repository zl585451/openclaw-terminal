# OCT v0.1.1 三平台发布指引

## ✅ 已完成

1. **跨平台日志修复**：ChatTab 不再使用 Windows 固定路径，Mac/Linux 上 Gateway 日志可正常显示
2. **版本号**：已升级为 `0.1.1`
3. **Windows 打包**：`release/OCT-Setup-v0.1.1.exe` 已生成

---

## 📋 你接下来要做的事

### 第 1 步：提交并推送代码

在 Cursor 终端执行：

```bash
git add .
git commit -m "v0.1.1: 跨平台日志修复 + 苏格拉底思维引导系统"
git push
```

### 第 2 步：打 tag 触发 Mac / Linux 自动构建

```bash
git tag v0.1.1
git push origin v0.1.1
```

推送后，GitHub Actions 会自动开始构建 Mac 和 Linux 版本。

### 第 3 步：打开 GitHub Release 页面

访问：`https://github.com/你的用户名/OpenClaw-Terminal/releases`

- 若已存在 `v0.1.1`，点击 **Edit**
- 若无，点击 **Draft a new release**，Tag 选择 `v0.1.1`

### 第 4 步：上传 Windows 安装包

将本地的 `E:\windows-window\OpenClaw-Terminal\release\OCT-Setup-v0.1.1.exe` 拖到 Release 的附件区域。

### 第 5 步：等待 Mac / Linux 构建完成

约 5–10 分钟后，Actions 会完成构建，以下文件会自动出现在同一 Release 中：

- `OCT-0.1.1-Mac-x64.dmg`（Intel Mac）
- `OCT-0.1.1-Mac-arm64.dmg`（M1/M2 Mac）
- `OCT-0.1.1-Linux-x64.deb` / `.rpm` / `.AppImage`

### 第 6 步：填写 Release 说明（可复制下方模板）
