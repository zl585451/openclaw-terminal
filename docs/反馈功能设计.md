# OCT 内置反馈功能设计

## 界面设计

```
┌─────────────────────────────────┐
│ 💬 使用感受如何？               │
├─────────────────────────────────┤
│                                 │
│ 😊 很好  | 😐 一般 | 😠 有问题  │
│                                 │
│ ┌─────────────────────────────┐│
│ │ 具体描述（可选）            ││
│ │                             ││
│ │                             ││
│ └─────────────────────────────┘│
│                                 │
│      [发送反馈]    [下次再说]   │
└─────────────────────────────────┘
```

## 实现方式

### 1. 添加到设置页面
在设置里添加"反馈"选项卡

### 2. 首次使用弹窗
用户首次启动时弹出

### 3. 定期提醒
每 7 天提醒一次（可关闭）

## 数据收集

收集内容：
- ✅ 满意度评分（1-5 星）
- ✅ 文字反馈（可选）
- ✅ 使用时长
- ✅ 系统信息（Windows/Mac/Linux）
- ❌ 不收集个人隐私

## 发送到哪里？

### 选项 A：GitHub Issues
自动创建 Issue

### 选项 B：飞书群
发送到飞书机器人

### 选项 C：邮箱
发送到少爷邮箱

### 选项 D：全部
同时发送到以上三个地方

---

**优点**：
- ✅ 用户体验好
- ✅ 反馈质量高
- ✅ 数据结构化

**缺点**：
- ❌ 需要修改 OCT 代码
- ❌ 需要重新打包发布
