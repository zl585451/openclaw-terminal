## OpenClaw 子代理对话系统 v2

基于意图识别的自动子代理调度系统 —— 让每个对话都独立执行

**版本**: 2.0  
**更新日期**: 2026-03-11  
**作者**: 少爷 & AMY

---

## 📖 项目概述

这套系统实现了所有用户对话自动 spawn 子代理处理的机制，主代理只做调度，子代理独立执行任务并自动汇报结果。

v2.0 全新升级：
- ✅ 四步判断法意图识别
- ✅ 适配中国大模型环境
- ✅ 完善的错误处理和超时管理
- ✅ 多任务并发控制
- ✅ 结果自动汇报规范

---

## 🎯 核心特性

| 特性 | 说明 |
|------|------|
| **自动 spawn** | 每条用户消息自动创建子代理 |
| **意图识别** | 四步判断法：对话/任务/歧义/耗时 |
| **并行执行** | 多个子代理可同时运行（最多 3 个） |
| **自动汇报** | 子代理完成后自动推送结果 |
| **会话保持** | 支持持久会话 (session) 和一次性任务 (run) |
| **错误隔离** | 子代理失败不影响主代理 |
| **超时管理** | 智能超时检测 + 用户提示 |
| **并发控制** | 任务排队机制，避免资源耗尽 |

---

## 🏗️ 架构设计

```
用户消息 → 主代理 (调度) → spawn 子代理 → 执行任务 → 自动汇报
    ↓
  不阻塞，可继续接收新消息
```

### 使用示例

**一次性任务**：
```javascript
sessions_spawn({
  runtime: "subagent",
  mode: "run",        // 完成后结束
  task: "查询上海天气",
  label: "天气查询"
})
```

**持久会话**：
```javascript
sessions_spawn({
  runtime: "subagent",
  mode: "session",    // 可继续对话
  task: "帮我开发一个功能",
  thread: true,
  label: "开发任务"
})
```

## 🧠 四步判断法（v2.0 新增）

### 步骤 1：这是一个任务，还是一次对话？
- **对话**（聊天、提问、解释）→ 直接回复，不 spawn
- **任务**（做某件事）→ 进入步骤 2

### 步骤 2：任务能在 30 秒内完成吗？
- **能**（心算、简单解释、格式转换）→ 直接回复，不 spawn
- **不能** → 进入步骤 3

### 步骤 3：任务需要调用工具吗？（搜索/读写文件/执行代码）
- **不需要** → 直接回复
- **需要** → spawn 子代理

### 步骤 4：任务是否有歧义？
- **有歧义**（缺少关键信息）→ 先澄清，再 spawn
- **清晰** → 立即 spawn

### 典型场景判断示例

| 用户说的话 | 意图 | 处理方式 |
|-----------|------|---------|
| "帮我解释一下 async/await" | 对话·解释 | 直接回复 |
| "帮我查一下今天的油价" | 任务·搜索 | spawn |
| "这个 bug 很烦" | 对话·吐槽 | 安慰，不 spawn |
| "把 utils.js 里的 TODO 修完" | 任务·写文件 | spawn |
| "帮我写个冒泡排序" | 任务·生成代码 | 直接回复（不需要工具） |
| "帮我把这 100 个文件都转成 JSON" | 任务·批处理 | spawn |
| "顺便查一下…" | 追加任务 | 单独 spawn |
| "算了不用了" | 取消 | 询问是否 kill 子代理 |

## ⚠️ 注意事项

- `thread=true` 仅当 channel 支持 subagent_spawning hooks 时可用
- 飞书等 channel 可能需要额外配置
- 子代理消耗独立 token 配额
- 同时最多 spawn **3 个**子代理（高延迟环境预留余量）

## 🎯 设计理念

| 原则 | 说明 |
|------|------|
| **主代理轻量化** | 只做调度，不执行具体任务 |
| **子代理专业化** | 每个子代理专注单一任务 |
| **结果自动推送** | 完成后主动汇报，无需轮询 |
| **错误隔离** | 子代理失败不影响主代理 |
| **用户友好** | 清晰的进度提示和错误说明 |
